# アレッピー（Alleppy） - 企画・計画書

## 概要

日本の飲食店の料理のアレルゲン情報に、誰でも・簡単に・速くアクセスできるアプリケーション。

## 背景・課題

### きっかけ

- パートナーが落花生・そばアレルギーを持っている
- バーミヤン（スカイラークグループ）で担々麺を注文し、落花生が含まれていることを知らずに口にしてしまった
- 一口で気づいたため軽傷で済んだが、そのまま食べ続けていたらアナフィラキシーショックの危険があった

### 現状の問題点

1. **アレルゲン情報へのアクセスが悪い**
   - バーミヤンの場合：タッチパネルの料理選択画面にアレルゲン表示なし
   - アレルゲン情報を見るには：アレルギー表示ボタン → QRコード表示 → スマホで読み込み → 専用ページ → PDFまたはフィルタ検索
   - この導線が面倒で、実際に確認しないユーザーが多い

2. **「調べるのが面倒」問題**
   - 情報は存在するがアクセスが悪すぎて調べない
   - 毎回調べる手間が心理的障壁になる
   - 店員に聞くのが苦手な人もいる

3. **「そもそも知らなかった」問題**
   - アレルギー選択の漏れや、選択項目に該当アレルギーがない場合
   - PDF一覧の見落とし
   - 個人の確認責任とされるが、命に関わる問題

4. **AIに聞いても100%の回答は得られない**
   - 命に関わる質問のため、AIは確定的な回答を避ける

## ソリューション

### コンセプト

アレルギーを持つ人が、飲食店の料理のアレルゲン情報を「調べる手間なく」把握できるアプリ。

### コア機能

1. **プロフィールにアレルギー登録**
   - 初回登録のみ。以降は自動でフィルタリングされる
   - 毎回アレルギーを選択する手間を排除

2. **3モード表示切り替え**
   - 安全な料理のみ表示
   - 危険な料理のみ表示
   - 全料理表示 + 危険な料理にアレルゲンマーク

3. **店舗検索・選択**
   - 事前検索（じっくり選べる）
   - 店内利用（速度とシンプルさ重視のUI）

4. **オフライン対応**
   - ローカルDBにデータを保持
   - オンライン時に定期同期
   - 震災時・回線障害時でも利用可能

5. **AI自然言語検索（将来実装）**
   - データベースを活用した自然言語ベースのアレルゲン検索
   - まずはデータベース基盤を優先して構築

## ターゲットユーザー

### 国内ユーザー（無料）
- 食物アレルギーを持つ人（成人約1-2%、子供約10%）
- アレルギーを持つ家族がいる人
- アレルギーを持つ友人・パートナーと食事する人

### インバウンドユーザー（有料）
- 訪日外国人観光客でアレルギーを持つ人
- 日本語が読めず、アレルゲン情報にアクセスできない人
- 年間訪日外国人3000万人超のうち、アレルギー持ち推定5%（約150万人）

## 技術方針

### 開発コスト
- **AI費用以外は完全0円** で開発・リリースする
- 無料サービス・無料枠のみで構成

### プラットフォーム
- **MVP**: PWA（Webアプリ）— Expo Webでビルド、Vercel/GitHub Pagesにデプロイ（0円）
- **将来**: ネイティブアプリ（iOS/Android）— 同じコードベースからExpo Buildで生成（ストア公開費用が必要になった時点で検討）

### データ管理（バックエンド不要）
- スクレイピングしたデータをJSONファイルとしてアプリにバンドル
- バックエンドサーバー・DB不要（MVP段階）
- ユーザーのアレルギー登録はlocalStorageに保存

### データ収集戦略
- **Phase 1**: 大手チェーンの公式公開データをPlaywrightでスクレイピング → JSON出力 → gitにcommit
- **Phase 2**: 中小チェーン・個人店へ拡大（掲載要望受付等）
- **テスト段階**: バーミヤン（スカイラークグループ）1社でプロトタイプ

### オフライン対応
- PWAのService Workerでオフラインキャッシュ
- アレルゲンデータはアプリにバンドル済みのため、オフラインでも完全動作

## 免責・注意表示

- 公式情報を元に作成していることを明示
- ダブルチェック（店舗での確認）を推奨
- アプリの情報のみに依存しないよう注意喚起

## アプリ名

**アレッピー（Alleppy）**
- アレルゲン + チャッピー（ChatGPT）の造語
- 覚えやすく親しみやすい語感
- 将来のAI機能との整合性がある
- 「っ」の二重子音を反映した英語表記（Alleppy）

## 開発ロードマップ

### Step 0: 企画・計画の整理 ← 完了
### Step 1: データ収集の調査
- バーミヤン（スカイラーク）のアレルゲンデータの形式・構造を確認
- スクレイピング可能性の検証
- データ収集方法の決定

### Step 2: アプリの設計
- 技術スタック選定（React Native vs Flutter等）
- DB設計
- 画面フロー設計
- UI/UXデザイン

### Step 3: MVP開発（PWA）
- バーミヤンのデータでプロトタイプ作成
- コア機能の実装（プロフィール登録、3モード表示、検索）
- PWA対応（Service Worker、オフラインキャッシュ）
- Vercelにデプロイ（0円）

### Step 4: データ拡充
- 大手チェーン全体へスクレイピング対象を拡大
- GitHub Actions cronでデータ更新の自動化

### Step 5: インバウンド対応
- 多言語対応（英語、中国語、韓国語等）
- 有料プランの実装

### Step 6: AI機能
- 自然言語ベースのアレルゲン検索
- データベースを活用したインテリジェント検索

### Step 7: ネイティブアプリ化（費用発生）
- 同じコードベースからExpo Buildでネイティブアプリ生成
- Apple Developer Program取得（$99/年） → App Store公開
- Google Play Developer取得（$25） → Play Store公開
- expo-sqlite導入によるローカルDB強化

## 参考情報

- **アレルギー表示ナビ**: 想定しているアレルギー検索とは方向性が異なる
- 競合は少ないか弱いと想定
