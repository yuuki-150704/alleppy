# アレッピー（Alleppy） - プロジェクトコンテキスト

## プロジェクト概要

日本の飲食店の料理のアレルゲン情報に、誰でも・簡単に・速くアクセスできるWebアプリ（PWA）。
将来的にネイティブアプリ（iOS/Android）へ展開予定。
詳細な企画・計画は `docs/planning.md` を参照。

## 開発方針

- **完全0円開発**（AI費用以外）: 無料サービスのみで構成
- **爆速開発**: AIを最大活用して最速でMVPリリース
- **PWAファースト**: Webアプリ（PWA）として先にリリース → 後からネイティブアプリ化
- **オープンソース的思想**: まず無料で公開し、問題提起と社会的インパクトを優先
- **データ収集**: 大手チェーンの公式公開データをスクレイピングで収集（Phase 1）
- **テスト対象**: バーミヤン（スカイラークグループ）を最初のテストデータとして使用

## 技術スタック（確定）

- **フレームワーク**: React Native (Expo) + TypeScript + Expo Router
- **ビルドターゲット**: Expo Web → PWA（ブラウザで動作、ホーム画面追加でアプリ化）
- **データ管理**: JSONファイルをアプリにバンドル（バックエンド不要）
- **ユーザーデータ**: AsyncStorage (localStorage) でアレルギー登録を保持
- **オフライン**: Service Worker でPWAオフライン対応
- **スクレイピング**: Playwright (Node.js) — allergy.skylark.co.jp がSPAのため必須
- **デプロイ**: Vercel（無料枠）or GitHub Pages（無料）
- **多言語**: i18n-js（将来）
- **将来**: AI自然言語検索機能を搭載予定
- **将来**: ネイティブアプリ化（同じコードベースからExpo Buildで生成）
- 詳細な設計は `docs/step2-design.md` を参照

## コア機能

1. プロフィールにアレルギー登録 → 自動フィルタリング
2. 3モード表示切り替え（安全のみ / 危険のみ / 全表示+マーク）
3. 店舗検索（事前検索 + 店内利用、UIは使い分け）
4. オフライン対応
5. 多言語対応（インバウンド向け、有料）

## 収益モデル

- 国内: 無料
- インバウンド: 多言語対応を日数指定の少額課金

## 免責

- 公式データ準拠を明示 + ダブルチェック推奨
